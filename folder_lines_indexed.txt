3 'Overview'
4 'This document lists every file and folder in the EQC frontend React application, derived directly from EQC_Frontend_Technical_Spec.md (v1.1, May 2026). Each entry explains what the file does, what it owns, and why it exists. Use this as the authoritative reference when generating, navigating, or reviewing the codebase.'
5 'The application is a React 18 / TypeScript SPA built with Vite, authenticated via Azure AD MSAL, and communicates exclusively with the EQC backend REST API. The folder structure follows feature-first grouping within src/ so that every page module is self-contained and shared code is clearly separated into components/, hooks/, api/, types/, and utils/.'

77 'API Layer — src/api/'
78 'All backend communication is centralised here. Each file targets one database entity or domain. Components and hooks never call Axios directly — they always go through these functions. This means API changes only require updating one file per entity.'
79 'File'
80 'Purpose'
81 'axiosInstance.ts'
82 'Axios instance with base URL from VITE_API_BASE_URL. Request interceptor: acquires MSAL token silently, attaches Authorization: Bearer {token} header. Response interceptor: normalises errors into ApiError shape. On 401, clears auth state and redirects to /login.'
83 'auth.api.ts'
84 'getMe(): Promise&lt;AuthUser&gt; — calls GET /api/auth/me. Used on app load to fetch the authenticated user&apos;s profile and role from the backend.'
